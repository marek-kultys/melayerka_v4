Interactions

"read more..." buttons

"Read more" buttons are visible only on portrait positioned mobiles (they have display property set to none by default, see button.read-more in CSS).

To make a "read more" button, create button element of class read-more: <button class="like-link read-more">read more...</button>

Now content that should be showed after clicking the button must be placed immediately after it, with class set to more: <p class="more">some content</p>

Any tag can be used with class more, but p seems to be semantically the best.

After click, the button is hidden and the element of class more is showed. This is all done by CSS on modern browsers. On old webkits JavaScript will set timeout with a callback that adds class more-visible to the element.

light off/on effect

To enable switching light on/off, add button of class toggle-light anywhere: <button class="toggle-light">press to see in dark</button>

The image that should be replaced with a glowing version needs to be of class glow. After light goes off, JavaScript will find image of the same name, just with added -dark before file extension, and load it. So, /any/path/image.jpg will be replaced with /any/path/image-dark.jpg.

Clicking the button will also toggle class 'dark' on body element, so CSS can do the job. To change styles, use body.dark ... selector.

Retina

Retina versions of images are loaded by retina.js library. It finds images of the same name, just with added @2x before file extension. So, if the original image is /any/path/image.jpg, retina.js will load /any/path/image@2x.jpg (if it exists).

Preloading

Glow versions of images, mobile dividers (for both light on and light off versions) are preloaded.

js/bundle.min.js

This file consist of concatenated and minified libraries: * jquery * prefixfree - to automate CSS prefixing * html5shiv-printshiv - to make HTML5 elements render and print in older IE.
